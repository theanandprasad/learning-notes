Context Management in Agent Harnesses

By Aparna Dhinakaran — Co-founder & CPO, Arize AI • Published Apr 27, 2026 • Saved May 31, 2026
AI Agents Context Window Compaction / Memory Harness Internals

Every agent harness runs into the same limit: the context window is too small for everything the model might want to remember. The real question is no longer what goes into the prompt — it's how the harness manages context over time.

The central bet

Every context decision encodes an assumption about model behavior. The key question: should the harness proactively constrain context usage, or trust the model to manage its own budget? Pi, OpenClaw, Claude Code, and Letta make different choices — but are converging on the same underlying pattern.

A deep-dive into component #2 of What is an Agent Harness. Drawn from two years building Alyx, Arize's in-product agent, plus a read of four real codebases.

1 · Large file reads — what to keep when a file won't fit

File reads make the problem concrete: when a file is larger than what fits in context, someone has to decide what to keep. All four support offset/limit pagination — but the caps and the layering differ.

Letta's MemFS — the standout

A git-backed memory filesystem where the agent's persistent memory lives as markdown. Files in system/ are pinned to the system prompt; files outside are visible by name+description in a tree but not loaded until read. The agent manages its own progressive disclosure — moving files in/out of system/, reorganizing, updating descriptions — and edits auto-commit to a git remote.

Where the designs converge

The most striking finding from comparing four codebases isn't how different they are — it's how much they agree. These aren't coincidences; they're convergent solutions to one problem: a fixed-size working set that has to feel infinite.

Same playbook, different domain: Alyx

Arize's Alyx (built for data exploration, not code) independently hit the same designs:

  • Caps tool results at a 10,000-token budget; binary-searches the largest dataset slice that fits
  • Deduplicates idempotent tool calls; keeps only the most recent preview
  • Splits big JSON into an LLM-visible preview + full server-side copy (drill in via jq) — same "persist outside context" pattern
  • Head+tail truncation with back-references to full content
  • char/4 token estimate; forces a checkpoint at 50,000 tokens where the model writes its own state summary before pruning

A product built for an entirely different domain converged on the same context-management playbook.

The closing analogy

The goal is not to show the model everything. It is to give it the right working set at the right time — and let it dynamically make decisions to manage its own context.
