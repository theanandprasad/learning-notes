What Is Agent Looping

By Shann³ (@shannholmberg) • Published Jun 8, 2026 • Saved Jun 9, 2026
AI Agents Agent Harnesses Looping Orchestration

For the last two years we prompted agents one task at a time. That's starting to change: instead of driving every step yourself, you set up a loop that handles discovery, planning, the work, checking, and iterating — until the goal is met.

The shift

Looping is a setup you build, not a prompt you send. Almost any agent harness can run it — it just depends on how you wire it up. You stop prompting each step; you define a goal and the system repeats the cycle until the work clears the requirements you set.

Here's your monthly reminder that you shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents.— Peter Steinberger (@steipete), the post Shann is quoting

The simplest loop: one agent working on itself

The bigger version: a fleet looping

You give an orchestrator agent a goal; it breaks the goal into pieces and hands each piece to a specialist agent; those specialists hand smaller jobs to their own subagents. The whole tree keeps looping through discovery, planning, execution, and verification until the goal is met.

Open vs. closed looping

Open looping — exploratory

Still has conditions and a goal, but you give the agent (or fleet) a wide space to move in. It can try different paths, discover things, build something you didn't fully spec out. "The exciting end" — where Shann wants to spend more time.

The catch: it burns an insane amount of tokens. For the 90% without an unlimited budget it isn't runnable yet — and pointed at projects with a loose standard, it "turns into a slop machine."

Closed looping — bounded

A human designs the end-to-end path first. The agents still loop, but inside the framework you built. It gets better every run (each pass feeds the next) and runs on a normal budget because the path is tight.

For most marketing work, closed is the one that pays off today.

What a closed loop is made of

A clear goal.
Defined steps.
An eval at each step.
A point where it stops or hands back to you — and feeds back performance data.
